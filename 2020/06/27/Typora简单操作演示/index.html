    通过常规的 markdown 语法和相对路径来引用图片和其它资源可能会导致它们在存档页或者主页上显示不正确，Hexo的官方文档中给出了一个解决方案：

    ①在根目录下配置文件_config.yml 中有 post_asset_folder:false改为true。这样在建立文件时，Hexo会自动建立一个与文章同名的文件夹，这样就可以把与该文章相关的所有资源（图片）都放到那个文件夹里方便后面引用。如这里我放了一张test.jpg的图片。

    【注】在Typora的偏好设置中选择此选项

    ②git bash安装插件：cnpm install https://github.com/7ym0n/hexo-asset-image –save（我是利用cnpm下载插件的，在我的博客文章[Hexo搭建个人博客][https://leo-fang.github.io/2020/06/27/Hexo%E6%90%AD%E5%BB%BA%E4%B8%AA%E4%BA%BA%E5%8D%9A%E5%AE%A2/ ]中有提到，也可以执行指令：npm install https://github.com/7ym0n/hexo-asset-image –save）
